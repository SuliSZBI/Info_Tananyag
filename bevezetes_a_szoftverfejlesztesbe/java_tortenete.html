A Java nyelv története, kialakulása

James Gosling, Mike Sheridan és Patrick Naughton 1991 júniusában indították el a Java nyelvi projektet. A Javát eredetileg interaktív televíziózásra tervezték, de akkoriban túl fejlett volt a digitális kábeltelevíziós iparág számára. A nyelvet kezdetben Oak-nak hívták, egy tölgyfáról, amely Gosling irodája előtt állt. Később a projekt a Green nevet kapta, végül pedig Java-ra nevezték át, a Java coffee, egy indonéz kávéfajta után. Gosling a Javát C / C++ stílusú szintaxissal tervezte, amelyet a rendszer- és alkalmazásprogramozók ismerősnek találtak.

A Sun Microsystems 1996-ban adta ki az első nyilvános implementációt Java 1.0 néven. Az „írd meg egyszer, futtasd bárhol” (WORA - Write Once Run Anywhere) funkcionalitást ígérte, költségmentes futási időt biztosítva a népszerű platformokon. Meglehetősen biztonságos és konfigurálható biztonsággal rendelkezett, lehetővé téve a hálózati és fájlhozzáférési korlátozásokat. A nagyobb webböngészők hamarosan beépítették a Java kisalkalmazások weboldalakon belüli futtatásának lehetőségét, és a Java gyorsan népszerűvé vált.

Eredeti alapelvek:

  • Egyszerűnek, objektumorientáltnak (object-oriented) és ismerősnek (familiar) kell lennie.
  • Robusztusnak (robust) és biztonságosnak kell lennie.
  • Felépítménysemlegesnek (architecture-neutral) és hordozhatónak (portable) kell lennie.
  • Nagy teljesítménnyel kell működnie.
  • Értelmezettnek (interpreted), szálkezeltnek (threaded) és dinamikusnak kell lennie.

A Java nyelv fejlődése

A Java 1.0 fordítót Arthur van Hoff írta át Java nyelvre, hogy szigorúan megfeleljen a Java 1.0 nyelvi specifikációnak. A Java 2 megjelenésével (eredetileg J2SE 1.2 néven jelent meg 1998 decemberében) az új verziók több konfigurációval rendelkeztek, amelyek különböző típusú platformokhoz készültek. A J2EE olyan technológiákat és API-kat tartalmazott, amelyek jellemzően szerverkörnyezetben futnak, míg a J2ME mobilalkalmazásokra optimalizált API-kat tartalmazott. Az asztali verziót J2SE-re nevezték át. 2006-ban marketing célokból a Sun átnevezte az új J2 verziókat Java EE-re , Java ME-re és Java SE- re.

1997-ben a Sun Microsystems megkereste az ISO/IEC JTC 1 szabványügyi testületet, majd később az Ecma Internationalt a Java formalizálása érdekében, de hamarosan kilépett a folyamatból. A Java továbbra is de facto szabvány, amelyet a Java Community Process szabályoz. Egy időben a Sun a legtöbb Java implementációját ingyenesen elérhetővé tette, a saját szoftver státuszuk ellenére.

2006. november 13-án a Sun kiadta Java virtuális gépének (JVM) nagy részét szabad és nyílt forráskódú szoftverként (FOSS- Free, Open-Source Software), a GPL-2.0 licenc feltételei szerint. 2007. május 8-án a Sun befejezte a folyamatot, és a JVM összes alapkódját elérhetővé tette szabad szoftver/nyílt forráskódú terjesztési feltételek mellett, eltekintve egy kis kódrészlettől, amelynek szerzői jogait nem a Sun birtokolta.

Az Oracle Corporation 2009-ben felvásárolta a Sun Microsystemst.

Java platformok

A Sun a Java négy, különböző alkalmazáskörnyezeteket célzó kiadását definiálta és támogatja, és számos API-ját szegmentálta úgy, hogy azok az egyik platformhoz tartozzanak. A platformok a következők:

  • Java kártya (Java Card) - okos kártyákhoz.
  • Java Platform, Micro Edition (Java ME) - korlátozott erőforrásokkal rendelkező környezeteket céloz meg.
  • Java Platform, Standard Edition (Java SE) - munkaállomás-környezeteket céloz meg.
  • Java Platform, Enterprise Edition (Java EE) - nagy, elosztott vállalati vagy internetes környezeteket céloz meg.
